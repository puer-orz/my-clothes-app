# 👔 本地个人 AI 穿搭衣橱 - 实现方案

## 1. 技术栈选型 (极简与本地化优先)
为了满足“Mac 本地”、“代码极简”、“脚本式运行”且具备“AI 能力”的需求，我们将采用以下技术栈：

*   **编程语言**: **Python** (Mac 系统完美支持，AI 开发首选)。
*   **应用框架**: **Streamlit** (极低代码量的 Web 应用框架，无需前端知识，界面美观，专为数据/AI 应用设计)。
*   **数据存储**: **SQLite** (Python 内置，无需安装，单文件存储，适合个人应用) + **本地文件系统** (存储图片)。
*   **AI 能力**:
    *   **核心推荐 (文字)**: 建议调用云端 API (如 DeepSeek/OpenAI)，这是实现“极低开发成本”且保证效果的唯一路径。若必须完全离线，可配合 **Ollama** (需用户额外安装)。
    *   **视觉呈现 (图片)**: 
        *   **基础版**: 自动拼图展示（将推荐的衣服图片组合展示）。
        *   **进阶版**: 调用绘图 API 生成“虚拟模特上身示意图”（注意：真实且精准的“AI 虚拟试穿”技术极其复杂，不符合本项目“极简”约束，我们将采用“生成佩戴效果示意图”代替）。
*   **天气数据**: **wttr.in** (开源天气服务，无需申请 Key，直接 HTTP 请求)。

## 2. 系统架构设计

### 2.1 目录结构
```text
my_wardrobe/
├── app.py              # 主程序入口 (Streamlit)
├── pages/              # 页面模块
│   ├── 1_👗_衣橱录入.py
│   ├── 2_👚_我的衣橱.py
│   ├── 3_🪄_今日穿搭.py
│   └── 4_⚙️_个人设置.py
├── utils/
│   ├── database.py     # 数据库操作 (SQLite)
│   ├── ai_service.py   # AI 接口封装 (LLM 调用)
│   └── helpers.py      # 工具函数 (天气、图片处理)
├── assets/             # 存放用户上传的衣服/身材图片
├── data/
│   └── wardrobe.db     # SQLite 数据库文件
├── requirements.txt    # 依赖库列表
└── .env                # 配置文件 (API Key)
```

### 2.2 核心数据模型 (SQLite)
1.  **Users**: 存储身高、体重、风格偏好、身材描述。
2.  **Clothes**: 
    *   `id`: 唯一编号
    *   `category`: 分类 (上衣/下装/鞋/配饰)
    *   `image_path`: 本地图片路径
    *   `description`: 风格/颜色/材质描述
    *   `size`: 尺码
    *   `season`: 适用季节
3.  **History**: 存储 AI 推荐的历史记录 (日期、推荐文案、搭配的衣服ID列表)。

## 3. 核心功能实现逻辑

### Phase 1: 基础设施搭建
1.  初始化 Python 环境与 Streamlit 项目。
2.  建立 SQLite 数据库连接，自动创建所需数据表。
3.  实现基础的侧边栏导航与页面路由。

### Phase 2: 衣橱管理功能
1.  **衣橱录入**:
    *   使用 `st.file_uploader` 上传图片。
    *   表单填写：分类、尺码、描述。
    *   保存图片到 `assets/`，元数据写入数据库。
2.  **我的衣橱**:
    *   从数据库读取数据。
    *   使用网格布局展示衣服图片卡片。
    *   支持按分类、季节筛选。

### Phase 3: AI 穿搭核心
1.  **个人设置**: 完善用户画像 (身高/体重/风格)。
2.  **获取环境数据**: 调用 `wttr.in` 获取当前 IP 所在地的天气、温度。
3.  **AI 推荐逻辑**:
    *   构建 Prompt: 包含“天气信息” + “用户画像” + “当前衣橱所有衣服的文本列表(ID+描述)”。
    *   请求 LLM: 让 AI 从衣橱中挑选合适的单品 ID，并生成推荐理由。
    *   解析返回: 提取衣服 ID，从本地库中通过 ID 找到对应的图片。
4.  **展示结果**:
    *   左侧: 显示 AI 生成的暖心推荐文案。
    *   右侧: 展示被推荐的衣服图片组合 (拼图)。
    *   (可选) 虚拟人生成: 根据用户描述生成一张 AI 模特图作为参考。

## 4. 实施步骤
1.  **环境准备**: 创建项目文件夹，安装依赖。
2.  **功能开发**: 依次实现数据库 -> 录入 -> 展示 -> AI 推荐。
3.  **本地运行**: 交付运行脚本，用户只需双击或终端运行命令。
